# Employee Office RAG System

Local office chatbot and admin app that combines:

- MySQL employee management tables for structured employee data.
- Qdrant vector search for company PDFs, DOCX files, TXT notes, and image OCR text.
- Ollama for local embeddings and LLM answers.
- LangChain + LangGraph for routing questions to SQL, Qdrant, or both.
- Multiple local FastAPI micro-APIs for health, employees, ingestion, retrieval, and chat.
- Streamlit UI for employee entry, document ingestion, and chat.

## 1. Prerequisites

Install and run these natively on Windows:

| Service | Install | Run |
|---------|---------|-----|
| MySQL 8 | [MySQL Installer](https://dev.mysql.com/downloads/installer/) | Start Windows service `MySQL80` on port `3306` |
| Qdrant | [Qdrant releases](https://github.com/qdrant/qdrant/releases), then extract `qdrant.exe` to `tools\qdrant\` | `.\scripts\start_qdrant.cmd` |
| Ollama | [ollama.com/download](https://ollama.com/download) | Runs automatically after install |

Recommended Ollama models:

```powershell
ollama pull llama3.1
ollama pull nomic-embed-text
```

Docker is optional for MySQL/Qdrant, but the default Windows flow uses native MySQL, bundled `qdrant.exe`, and local Ollama. If PowerShell blocks `.ps1` scripts, use the matching `.cmd` script.

Optional Docker fallback for MySQL and Qdrant:

```powershell
.\scripts\start_docker_dependencies.cmd
```

Ollama still needs to be installed locally unless you intentionally run your own Ollama server and update `OLLAMA_BASE_URL` in `.env`.

## 2. Setup

```powershell
cd D:\ADS1
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
```

Edit `.env` and set your MySQL username/password. MySQL 8's default authentication requires `cryptography`, which is included in `requirements.txt`.

## 3. Start MySQL and Qdrant

1. Start MySQL from Windows Services: `MySQL80`.
2. Start Qdrant:

```powershell
.\scripts\start_qdrant.cmd
```

3. Verify dependencies and create tables:

```powershell
.\scripts\start_dependencies.cmd
```

This waits for local MySQL/Qdrant, runs `scripts\init_db.py`, and prints a system check.

## 4. Create MySQL Tables Manually

Option A: Open MySQL Workbench, paste `sql\schema.sql`, and run it.

Option B: Run from Python:

```powershell
python scripts\init_db.py
```

## 5. Run APIs

Start all local micro-APIs:

```powershell
.\scripts\start_apis.cmd
```

| Service | Port | Docs |
|---------|------|------|
| Health & diagnostics | 8000 | http://127.0.0.1:8000/docs |
| Employees CRUD | 8001 | http://127.0.0.1:8001/docs |
| Ingestion upload/text | 8002 | http://127.0.0.1:8002/docs |
| Retrieval search | 8003 | http://127.0.0.1:8003/docs |
| Chat RAG | 8004 | http://127.0.0.1:8004/docs |

Optional all-in-one gateway on a single port:

```powershell
uvicorn app.api:app --reload --host 127.0.0.1 --port 8090
```

Set `API_*_URL` values in `.env` if you use non-default ports.

## 6. Run Streamlit App

The Streamlit UI calls every micro-API. Start all APIs first, then Streamlit:

```powershell
streamlit run streamlit_app.py
```

Or start everything locally:

```powershell
.\scripts\run_local.cmd
```

## 7. Ingest Company Documents

You can upload documents from the Streamlit app or put files into `data\documents` and run:

```powershell
python scripts\ingest_folder.py data\documents
```

Supported by default:

- `.pdf`
- `.docx`
- `.txt`
- `.md`
- `.png`, `.jpg`, `.jpeg` if Tesseract OCR is installed and available

## Architecture

```text
Streamlit (8501)
    -> Health API (8000)
    -> Employees API (8001)   - employee CRUD, office, salary, attendance, documents
    -> Ingestion API (8002)   - upload/text into Qdrant
    -> Retrieval API (8003)   - knowledge search
    -> Chat API (8004)        - RAG (sql / vector / both routes)
         -> MySQL + Qdrant + Ollama
```

Every UI action uses HTTP to these APIs only.

## How The Router Works

The LangGraph flow classifies each question:

- Employee names, salaries, departments, attendance, leave, joining date, IDs, emails, phone numbers: MySQL.
- Company policies, handbook/theory, uploaded PDFs/DOCX/images: Qdrant.
- Mixed questions such as "What is Varshith's salary and what does the leave policy say?": both.

The final answer is generated by Ollama from the retrieved context. If Ollama is not running, the chat API returns the retrieved SQL/document context with a clear message instead of crashing.
